# Spoken — 语音输入 AI 优化工具

> Last updated: 2026-04-04

---

## 1. 产品概述

**一句话定位：**
把脑子里的语音快速转化成高质量文本的菜单栏工具。

**核心场景：**
1. **文本优化** — 语音 → 纠错 + 润色 → 干净文本 → 剪贴板 → 粘贴使用
2. **Prompt 优化** — 语音 → 结构化 Prompt → 剪贴板 / 直接发给 AI

**目标用户：**
- 经常在电脑上打字，但懒得动手的人
- 想用语音高效输入长文本的人
- 需要把模糊想法快速转化成结构化 Prompt 的人

---

## 2. 功能规划

### 2.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 语音录制 | 点击/快捷键开始录制，松开结束 | P0 |
| 语音识别 | 使用 Mac 原生听写（SFSpeechRecognizer，离线） | P0 |
| 文本优化 | MiniMax API，纠错 + 润色 + 语气调整 | P0 |
| Prompt 优化 | MiniMax API，重构为结构化 AI Prompt | P0 |
| 文本输入 | 优化后文本直接输入到当前焦点窗口（模拟键盘） | P0 |
| 快捷键触发 | 全局快捷键，任意界面可开始说话 | P1 |
| 模式切换 | 「文本模式」和「Prompt 模式」切换 | P1 |

### 2.2 非功能性

- 完全本地执行（语音识别离线，MiniMax API 调用走网络）
- 响应速度：录音→识别→优化 全流程 < 5 秒
- 菜单栏常驻，不占 Dock

## 3. 技术方案

### 架构
```
麦克风 → SFSpeechRecognizer（本地）→ 原始文本
    ↓
[两个分支]
    ├→ MiniMax 文本优化 API → 模拟键盘输入 → 当前焦点窗口
    └→ MiniMax Prompt 优化 API → 模拟键盘输入 → 当前焦点窗口
```

### 技术栈
- **语言：** Swift（原生 macOS App）
- **语音识别：** SFSpeechRecognizer
- **AI 调用：** MiniMax API（abab6.5s 或 homo）
- **UI：** SwiftUI 菜单栏 App（LSUIElement = 不占 Dock）
- **快捷键：** Swift 系统快捷键框架或 HotKey.swift

### 项目结构
```
spoken/
├── Spoken/              # 主应用
│   ├── App/
│   ├── Views/
│   ├── Services/
│   │   ├── SpeechService.swift      # 语音识别
│   │   ├── MiniMaxService.swift     # MiniMax API
│   │   └── ClipboardService.swift   # 剪贴板
│   └── Models/
├── project.yml          # XcodeGen 配置
└── PRD.md
```

## 4. 详细设计

### 4.1 交互流程

**方式一：菜单栏点击**
1. 点击菜单栏图标 → 弹出面板
2. 点击「开始说话」或按快捷键
3. 说话完成后自动处理
4. 优化结果直接输入到当前焦点窗口，通知用户

**方式二：快捷键**
1. 全局快捷键（默认 ⌥ + V）
2. 呼出浮动窗口，开始录制
3. 松开快捷键或点击完成
4. 结果直接输入到焦点窗口，窗口消失

### 4.2 MiniMax Prompt 设计

**文本优化 Prompt：**
> 你是一个文字优化助手。用户输入了一段语音转文字的内容，可能有错字、口水话、语气不通顺。请帮他润色成通顺、简洁、得体的中文文本，保持原意。如果有明显错误请纠正，但不要过度修改。

**Prompt 优化 Prompt：**
> 你是一个 Prompt 工程助手。用户的输入是他想对 AI 说的话，请把它重构为一个结构清晰、意图明确、包含必要上下文的 AI Prompt。要求：明确任务目标、提供必要的输入信息、说明输出格式（如需要）。
