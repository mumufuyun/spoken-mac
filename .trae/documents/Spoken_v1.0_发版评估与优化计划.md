# Spoken v1.0 发版评估与优化计划

## 一、当前状态评估

### 已跑通的核心流程
- ✅ 快捷键触发 (⌥+空格)
- ✅ 语音识别 (Apple SFSpeechRecognizer)
- ✅ 6 种 AI 后处理模式 (直接输入/润色/Prompt/翻译/摘要/格式化)
- ✅ 文本注入 (剪贴板 + osascript + CGEvent)

### 已知问题清单

| 优先级 | 问题 | 影响 |
|--------|------|------|
| **P0** | 偶发性语音识别失败（约 10-20% 失败率） | 用户首次或某次录音完全无内容 |
| **P1** | MiniMax API 超时（10秒超时仍可能触发） | AI 后处理失败，回退原始文本 |

## 二、优化方案

### 方案一：P0 — 语音识别稳定性（最高优先级）

**问题：** `AVAudioEngine` 的 tap 偶发性注册失败，导致录音无内容

**解决方案：**

1. **增加首次录音延迟保护**
   - 在 `audioEngine.start()` 后等待 200ms 再允许录音
   - 给系统足够时间初始化音频管道

2. **增强重试机制**
   - 已有：1 秒后检测 `audioReceived`，最多重试 3 次
   - 优化：增加重试间隔（100ms → 200ms → 500ms），给系统更多恢复时间

3. **失败提示用户**
   - 如果 3 次重试后仍无音频，浮窗显示"麦克风异常，请重试"
   - 而不是静默返回空文本

**实现步骤：**
```
SpeechService.swift:
  - startRecording() → 增加 200ms 启动缓冲
  - installTapAndStart() → 重试间隔改为递增
  - 新增 audioCheckTimeout() 处理最终失败
```

---

### 方案二：P1 — 降低 API 超时率

**问题：** MiniMax API 偶尔超时（10秒 → 已改为 60秒，但仍可能超时）

**解决方案：**

1. **增加请求重试**
   - API 超时/失败时自动重试 1 次
   - 重试间隔 1 秒

2. **降低 temperature**
   - 当前 `temperature: 0.3`，可适当降低以加快响应速度
   - 改为 `0.1`

3. **增加超时提示**
   - 如果 API 超时，浮窗显示"AI 服务响应慢，请稍后重试"
   - 而不是静默失败

**实现步骤：**
```
MiniMaxService.swift:
  - chat() → 增加 retry 逻辑
  - temperature 改为 0.1
  
AppDelegate.swift:
  - processAndInput() → 超时错误单独处理，显示提示
```

---

### 方案三：P1 — 提升整体处理速度

**问题：** 从录音结束到文本注入完成，用户等待约 3-8 秒

**分析时间分配：**
- 音频引擎停止 + 缓冲：~0.5s
- MiniMax API 调用：~2-5s（最大瓶颈）
- 目标应用激活：~0.5-1s
- 文本注入：~0.2-0.5s

**优化方案：**

1. **提前激活目标应用**
   - 在 AI 处理的同时激活目标应用（并行而非串行）
   - 可节省 0.5-1s

2. **减少等待延迟**
   - `onComplete` 中的 `0.5s + 1.0s` 延迟可优化为事件驱动
   - 检测应用已在前台后再注入，而非固定等待

3. **降低静音阈值**
   - 当前 5 秒 → 改为 3 秒
   - 用户说话后更快自动结束

4. **API 选择优化**
   - 当前 `MiniMax-M2.5-HighSpeed` 已经是高速版
   - 如果仍有延迟，可考虑换用更小的模型

**实现步骤：**
```
AppDelegate.swift:
  - onComplete 回调 → 并行激活应用和 AI 处理
  - 减少固定延迟，改用事件驱动
  
SpeechService.swift:
  - silenceThreshold 改为 3.0
```

---

### 方案四：P2 — 长语音内容兜底保护（核心需求）

**问题：** 用户录制了一段长语音（如 5 分钟），如果 AI 处理或注入失败，内容完全丢失

**解决方案：**

1. **AI 处理失败时保留原始文本**
   - 当前：AI 失败后直接回退原始文本注入 → ✅ 已有
   - 优化：增加系统通知提示用户"AI 处理失败，已使用原始文本"

2. **注入失败时复制到剪贴板**
   - 如果所有注入方法都失败，弹窗提示"文本已复制到剪贴板"
   - 确保用户不会丢失内容

3. **增加历史记录功能**
   - 每次录音的最终文本保存到本地文件
   - 用户可以从历史记录中找回丢失的内容

**实现步骤（分阶段）：**

**Phase 1（紧急）— 注入失败兜底：**
```
TextInjectionEngine.swift:
  - typeText() → 失败时弹窗提示 + 保留剪贴板
  - 增加"复制成功"通知
  
AppDelegate.swift:
  - processAndInput() → 注入失败时显示提示
```

**Phase 2（可选）— 历史记录：**
```
新增 HistoryService.swift:
  - 保存每次处理的文本到 ~/Library/Application Support/Spoken/history.json
  - 提供简单的历史记录查看界面
```
