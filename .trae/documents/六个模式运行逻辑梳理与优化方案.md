# 六个模式运行逻辑梳理与优化方案

## 一、当前六个模式完整分析

### 1. 直接输入（direct）
- **浮窗行为**：点击按钮，`mode` 设为 `.direct`，保存 `"直接输入"` 到 UserDefaults
- **录音流程**：AppDelegate `processAndInput` 读取 `"直接输入"` → 跳过 AI 处理 → 直接注入
- **Prompt**：无
- **问题**：正常工作

### 2. 润色（polish）
- **浮窗行为**：点击按钮，`mode` 设为 `.polish`，保存 `"润色"` 到 UserDefaults
- **录音流程**：AppDelegate 读取 `"润色"` → 映射为 `AIProcessingMode.polish` → 调用 MiniMaxService `processText` → `polish()` → AI 润色后注入
- **Prompt**：
  ```
  你是一个文字优化助手。用户输入了一段语音转文字的内容，可能有：
  1. 错别字 2. 标点符号缺失或断句混乱 3. 重复表达 4. 口语化表达
  请优化处理，只输出优化后的文本，不要解释，不要加引号，不要加前缀。
  ```
- **问题**：
  - ContentView 有自己的 `processAndInput` 但永远不会被调用（浮窗没有录音按钮了）
  - 实际处理完全走 AppDelegate 的 `RecordingViewModel.processAndInput`
  - 两套 `processAndInput` 逻辑重复且不一致

### 3. Prompt
- **浮窗行为**：点击按钮，`mode` 设为 `.prompt`，保存 `"Prompt"` 到 UserDefaults
- **录音流程**：AppDelegate 读取 `"Prompt"` → 映射为 `aiMode = .none`（直接返回原文！）→ 完全不做 Prompt 生成
- **问题**：Prompt 模式**完全失效**，映射为 `.none` 等于直接注入

### 4. 翻译（translate）
- **浮窗行为**：点击按钮显示语言选择（英/日/韩），保存 `"翻译"` 到 UserDefaults
- **录音流程**：AppDelegate 读取 `"翻译"` → 映射为 `AIProcessingMode.translate` → 调用 MiniMaxService `processText` → `translate(text:)` 固定翻译为英文（**忽略浮窗的语言选择！**）
- **问题**：浮窗的语言选择完全无效，AppDelegate 走的翻译路径是写死英文的

### 5. 摘要（summarize）
- **浮窗行为**：点击按钮，`mode` 设为 `.summarize`，保存 `"摘要"` 到 UserDefaults
- **录音流程**：AppDelegate 读取 `"摘要"` → 映射为 `AIProcessingMode.summarize` → 调用 MiniMaxService `processText` → `summarize()`
- **Prompt**：提取关键信息，生成简洁摘要
- **问题**：基本正常

### 6. 格式化（format）
- **浮窗行为**：点击按钮，`mode` 设为 `.format`，保存 `"格式化"` 到 UserDefaults
- **录音流程**：AppDelegate 读取 `"格式化"` → 映射为 `AIProcessingMode.format` → 调用 MiniMaxService `processText` → `format()`
- **Prompt**：整理成 bullet points、编号列表等结构化格式
- **问题**：基本正常

## 二、核心问题总结

### 问题 1：Prompt 模式完全失效
AppDelegate 的映射将 `"Prompt"` → `aiMode = .none`，导致直接注入原文。

### 问题 2：翻译语言选择无效
浮窗可以选择目标语言（英/日/韩），但 AppDelegate 走 `MiniMaxService.processText(mode: AIProcessingMode.translate)` → 固定翻译为英文。

### 问题 3：两套 processAndInput 逻辑重复
- `ContentView.processAndInput` 使用 `SpokenMode` 和 `TranslateLanguage`，调用 `MiniMaxService.process(text:mode:translateLang:)`
- `AppDelegate.RecordingViewModel.processAndInput` 使用字符串映射到 `AIProcessingMode`，调用 `MiniMaxService.processText(mode:)`
- 实际只有 AppDelegate 的在运行，ContentView 的代码是死代码

### 问题 4：AIProcessingMode 和 SpokenMode 两套枚举不一致
- `AIProcessingMode`：none/polish/translate/summarize/format（AppDelegate）
- `SpokenMode`：direct/polish/prompt/translate/summarize/format（ContentView）
- 两套系统映射混乱

## 三、优化方案

### 核心思路：统一使用 SpokenMode + 删除 AIProcessingMode

**步骤 1：删除 AIProcessingMode 枚举**
- 移除 AppDelegate 中的 `AIProcessingMode` 枚举
- 统一使用 ContentView.swift 中的 `SpokenMode`

**步骤 2：扩展 MiniMaxService 的统一入口**
```swift
// 当前已有的方法，支持 SpokenMode + TranslateLanguage
func process(text: String, mode: SpokenMode, translateLang: TranslateLanguage, completion: @escaping (Result<String, Error>) -> Void)
```

**步骤 3：修复 Prompt 模式映射**
AppDelegate 中 `"Prompt"` → 应映射为调用 `toPrompt()` 而不是 `.none`

**步骤 4：修复翻译语言传递**
- `RecordingViewModel` 增加 `translateLang` 状态
- 从 UserDefaults 读取翻译语言选择
- 传递给 MiniMaxService 的 `process(text:mode:translateLang:)` 方法

**步骤 5：清理 ContentView 的死代码**
- 移除 `processAndInput`、`injectText`、`checkPermissions` 等方法（ContentView 只是 UI 展示，不执行实际处理）
- ContentView 只负责：模式选择 + 保存到 UserDefaults + 状态展示

### 修改后完整流程

```
用户点击浮窗模式按钮
  → 保存 SpokenMode.rawValue 到 UserDefaults["spokenMode"]
  → 保存 TranslateLanguage.rawValue 到 UserDefaults["translateLang"]（仅翻译模式）

用户按 ⌥+空格 录音
  → AppDelegate.RecordingViewModel 读取 UserDefaults
  → 录音结束 → processAndInput
  → 根据 spokenMode 调用 MiniMaxService.process(text:mode:translateLang:)
  → 获得结果 → 文本注入
```

### 具体代码改动

1. **AppDelegate.swift**：
   - 删除 `AIProcessingMode` 枚举
   - `RecordingViewModel` 增加 `translateLang` 读取
   - `processAndInput` 改为直接调用 `MiniMaxService.shared.process(text:mode:translateLang:)`
   - 删除字符串映射逻辑

2. **MiniMaxService.swift**：
   - `process(text:mode:translateLang:)` 已经完整支持所有模式（含 Prompt）
   - 删除多余的 `processText(mode:AIProcessingMode)` 方法
   - 保留 `translate(text:to:)` 和 `translate(text:)`（前者带语言参数）

3. **ContentView.swift**：
   - 删除 `processAndInput`、`injectText`、`checkPermissions` 方法
   - 保留模式按钮 + UserDefaults 保存
   - 保留翻译语言选择 + UserDefaults 保存
   - 保留权限检查（仅用于 UI 显示状态）
